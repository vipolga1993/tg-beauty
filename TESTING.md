# TESTING.md — Руководство по тестированию

## 1. О проекте

Telegram Mini App для онлайн-записи к бьюти-мастеру Анне Ивановой.
Стек: HTML + CSS + vanilla JS. Без сборки, без фреймворков.

**Бот:** `@tg_beautybot`
**Продакшн:** https://tg-app-tan.vercel.app
**Деплой:** Vercel (из папки `tg-app/`)

---

## 2. Как запустить локально

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd tg-beauty

# 2. Запустить локальный сервер
npx serve tg-app -l 3000

# 3. Открыть в браузере
http://localhost:3000
```

Приложение работает в браузере без Telegram.
Telegram API (haptic, BackButton, sendData) будут недоступны — это нормально, есть фоллбэки.

## 6. localStorage-ключи

| Ключ | Значение | Назначение |
|------|----------|------------|
| `welcome_done` | `'1'` | Онбординг показан, больше не показывать |
| `offer_shown_v2` | `'1'` | Модалка скидки показана, больше не показывать |

**Сброс для повторного тестирования** (в консоли браузера):
```js
localStorage.removeItem('welcome_done');
localStorage.removeItem('offer_shown_v2');
location.reload();
```

Или полная очистка:
```js
localStorage.clear();
location.reload();
```

## 9. Анимации

| Элемент | Анимация | Длительность |
|---------|---------|--------------|
| Переход экранов | Slide + fade (translateX ±40px) | 250ms |
| Карточки услуг, портфолио, даты, слоты | fadeIn (каскад) | 300ms + задержка 40ms на элемент |
| Зелёная галочка (экран успеха) | bounceIn (scale 0.3 → 1.08 → 1) | 450ms |
| Модалка-оффер (overlay) | Opacity 0 → 1 | 300ms |
| Модалка-оффер (карточка) | translateY(30px) scale(0.95) → 0 | 350ms, spring-кривая |
| Кнопки при нажатии | scale(0.97) + opacity 0.9 | 100ms |
| Точки карусели (active) | scale(1.2) + opacity | 200ms |

**Что проверить:**
- [ ] Переход вперёд — слайд справа
- [ ] Переход назад — слайд слева
- [ ] Модалка появляется плавно, а не мгновенно
- [ ] Галочка «пружинит» при появлении
- [ ] Карточки услуг появляются каскадом сверху вниз

## 12. E2E-сценарии

### Сценарий A: Полный путь записи (первый визит)

1. Открыть приложение (localStorage пуст)
2. **→ Онбординг** — проверить текст, нажать «Начать»
3. **→ Каталог** — проверить профиль, портфолио, 7 услуг
4. **→ Модалка-оффер** — проверить текст, нажать «Пропустить»
5. Нажать на «Маникюр классический» (1 500 ₽, 60 мин)
6. **→ Детали** — проверить данные, нажать «Записаться»
7. **→ Дата/время** — выбрать ближайший будний день
8. Убедиться: 12 слотов, кнопка «Записаться» скрыта
9. Выбрать 14:30
10. Убедиться: карточка-сводка и кнопка «Записаться» видны
11. Нажать «Записаться»
12. **→ Успех** — проверить карточку: 💅, дата, 14:30–15:30, 1 500 ₽
13. Нажать «Вернуться в каталог»
14. **→ Каталог** — состояние сброшено

### Сценарий B: Повторный визит

1. Открыть приложение (localStorage заполнен)
2. Онбординг НЕ показывается → сразу каталог
3. Модалка-оффер НЕ показывается
4. Записаться на другую услугу

### Сценарий C: Навигация «Назад»

1. Каталог → Детали → Назад → Каталог
2. Каталог → Детали → Дата → Назад → Детали → Назад → Каталог
3. Убедиться: анимация «назад» — слайд слева

### Сценарий D: Смена даты после выбора времени

1. Выбрать дату, выбрать время 10:00
2. Сменить дату
3. Убедиться: выбор времени сброшен, кнопка «Записаться» скрыта

### Сценарий E: Наращивание ногтей (длительная услуга)

1. Выбрать «Наращивание ногтей» (150 мин = 2,5 ч)
2. Записаться на 15:00
3. Проверить расчёт: 15:00–17:30

### Сценарий F: Ремонт ногтя (короткая услуга)

1. Выбрать «Ремонт ногтя» (20 мин, 300 ₽)
2. Записаться на 17:30
3. Проверить расчёт: 17:30–17:50

## 13. Известные ограничения

1. **Кнопки «Позвонить» / «Написать»** — только haptic, никуда не ведут
2. **Портфолио** — заглушки (градиенты + эмодзи), не реальные фото
3. **Карусель на деталях** — 5 градиентных слайдов для всех услуг, не уникальные
4. **Слоты времени** — одинаковые для всех дат, нет реальной проверки занятости
5. **sendData** — отправляет данные боту, но бот не обрабатывает ответ (нет бэкенда)
6. **Команды /start, /help, /contact** — зарегистрированы в меню, но бот на них не отвечает
7. **Статистика** «247 записей · Ближайшее: завтра, 10:00» — захардкожена

## 14. Настройка бота (для справки)

Скрипт `setup-bot.js` настраивает бота через Telegram Bot API.
Токен хранится в `.env` (файл `BOT_TOKEN=...`).

```bash
node setup-bot.js
```

Устанавливает:
- Описание бота (setMyDescription)
- Короткое описание (setMyShortDescription)
- Команды: /start, /help, /contact (setMyCommands)
- Кнопка меню «Записаться» → открывает Mini App (setChatMenuButton)
