# Бриф: Telegram Mini App — каталог услуг бьюти-мастера

## Концепция

Клиент открывает Telegram-бота → нажимает кнопку «Наши услуги» → открывается Mini App с каталогом услуг → выбирает услугу → нажимает «Записаться» → данные уходят боту → бот подтверждает запись.

**Принципы:**
- Выглядит как нативное приложение, не как сайт
- Максимум визуала, минимум текста
- До записи — максимум 2 нажатия
- Русский язык интерфейса
- Крупные тап-зоны (минимум 44×44px)
- Адаптация под тему Telegram (светлая/тёмная)


**Элементы на экране:**

| Элемент | Тип | Описание |
|---|---|---|
| Иконка ✓ | Иконка/анимация | Визуальный акцент |
| Заголовок | Текст | «Вы записываетесь на:» |
| Карточка-итог | Блок | Название услуги, цена, длительность |

| Пояснение | Текст, серый | «Мастер свяжется с вами для выбора даты и времени.» |
| MainButton | Telegram MainButton | «ПОДТВЕРДИТЬ ЗАПИСЬ» |

**Действия пользователя:**
- Нажатие **BackButton** (←) → возврат на **Экран 2**
- Нажатие **MainButton** «ПОДТВЕРДИТЬ ЗАПИСЬ» →
  - `Telegram.WebApp.sendData(JSON.stringify({ serviceId, serviceName, price, userId }))` — отправка данных боту

**Поведение Telegram API:**
- `BackButton.show()`
- `MainButton.setParams({ text: 'ПОДТВЕРДИТЬ ЗАПИСЬ', color: themeParams.button_color })`
- `MainButton.showProgress(true)` — при отправке данных
- `Telegram.WebApp.close()` — после успешной отправки

## Стилизация и дизайн

### Цвета — из Telegram themeParams

```css
:root {
  --bg: var(--tg-theme-bg-color);
  --text: var(--tg-theme-text-color);
  --hint: var(--tg-theme-hint-color);
  --link: var(--tg-theme-link-color);
  --button: var(--tg-theme-button-color);
  --button-text: var(--tg-theme-button-text-color);
  --secondary-bg: var(--tg-theme-secondary-bg-color);
}
```

### Типографика
- Заголовок услуги: 17px, font-weight 600
- Цена: 17px, font-weight 700, цвет `--link` или `--button`
- Длительность: 14px, цвет `--hint`
- Описание: 15px, цвет `--text`
- Системный шрифт: `-apple-system, 'SF Pro', 'Segoe UI', Roboto, sans-serif`

### Карточка услуги
- Фон: `--secondary-bg`
- Скругление: 12px
- Тень: нет (Telegram-стиль — плоский)
- Отступы: 16px между карточками
- Фото: скругление сверху 12px, aspect-ratio 16/9
- Внутренние отступы текста: 12px

### Кнопки и тап-зоны
- Минимум 44px высота
- Карточка целиком — одна тап-зона
- Эффект нажатия: `transform: scale(0.98)` + `transition: 0.1s`

## Telegram API — что используем

| API | Где | Зачем |
|---|---|---|
| `WebApp.ready()` | Инициализация | Сообщить Telegram, что приложение загрузилось |
| `WebApp.expand()` | Инициализация | Развернуть Mini App на полный экран |
| `WebApp.MainButton` | Экраны 2, 3 | Кнопка «Записаться» / «Подтвердить» |
| `WebApp.BackButton` | Экраны 2, 3 | Навигация назад |
| `WebApp.HapticFeedback` | Экраны 1, 2, 3 | Тактильная отдача при нажатиях |
| `WebApp.themeParams` | Везде | Цвета интерфейса |
| `WebApp.initDataUnsafe.user` | Экран 3 | Имя клиента для подтверждения |
| `WebApp.sendData()` | Экран 3 | Отправка заявки боту |
| `WebApp.close()` | Экран 3 | Закрытие после отправки |
| `WebApp.showPopup()` | При ошибках | Нативные диалоги ошибок |

## Бот: минимальная логика

1. **Команда /start** → приветственное сообщение + кнопка «Наши услуги» (WebApp button)
2. **Получение web_app_data** → парсинг JSON → сообщение клиенту: «Вы записались на {услуга}, {цена} ₽. Мастер свяжется с вами!»
3. **Уведомление мастеру** → пересылка заявки в личку мастера: «Новая запись! {имя клиента} (@username) хочет записаться на {услуга}»

## Чего НЕ будет в первой версии (MVP)

| Функция | Почему не сейчас | Когда добавим |
|---|---|---|
| Календарь с выбором даты/времени | Усложняет MVP, нужен бэкенд с расписанием | v2 |
| Онлайн-оплата | Нужен эквайринг, юр. вопросы | v2 |
| Категории/табы услуг | Для 5-10 услуг не нужны, достаточно скролла | v1.1 |
| Портфолио/галерея | Отдельный экран усложнит, достаточно фото на карточках | v1.1 |
| Отзывы клиентов | Нужна модерация, сбор отзывов | v1.1 |
| Скидки/промокоды | Усложняет логику, можно указать в описании | v1.1 |
| Админка для мастера | Пока обновляем JSON вручную | v2 |
| История записей | Нужна база данных | v2 |
| Поиск по услугам | Для 5-10 услуг не нужен | v2 |
| Мультиязычность | Только русский для старта | v2 |
| Push-напоминания | Нужна серверная логика с крон-задачами | v1.1 |
| Skeleton-загрузка | Данные из JSON грузятся мгновенно | v1.1 |
| Фото до/после | Достаточно одного фото на карточке для MVP | v1.1 |

## Технический стек

| Компонент | Технология | Почему |
|---|---|---|
| Frontend | React + Vite | Быстрая сборка, малый бандл, компонентный подход |
| Стилизация | CSS Modules + CSS-переменные Telegram | Нативный вид, нет лишних зависимостей |
| Данные | JSON-файл в проекте | Нет бэкенда для MVP |
| Бот | Node.js + grammY | Лёгкий, хорошая документация, TypeScript |
| Хостинг Mini App | GitHub Pages / Vercel | Бесплатно, HTTPS из коробки |
| Хостинг бота | Railway / Render | Бесплатный tier, автодеплой |
