# BACKEND-PLAN.md — SaaS-платформа для бьюти-мастеров

## Концепция

Мульти-тенантная SaaS-платформа внутри Telegram. Любой бьюти-мастер регистрируется через бота, получает персональный каталог (Mini App), настраивает услуги/портфолио/расписание через админку (Mini App), а клиенты записываются через deep link мастера.

**Модель монетизации:** Freemium + ЮKassa

| | FREE | PRO (299 руб/мес или 2 490 руб/год) |
|---|---|---|
| Услуги | до 5 | без ограничений |
| Фото портфолио | до 5 | без ограничений |
| Тема оформления | стандартная | кастомная (цвет, стиль) |
| Напоминания клиентам | нет | за 24ч и 2ч |
| Пересылка сообщений клиентов | нет | да |
| Статистика | базовая (кол-во записей) | расширенная (доход, популярные услуги) |

## 1. Архитектура

### Один бот + deep links

```
Клиент получает ссылку от мастера:
  t.me/tg_beautybot?start=m_anna_ivanova_5442
                              ^^^^^^^^^^^^^^^^
                              slug мастера

Бот получает /start m_anna_ivanova_5442
  → открывает Mini App: https://tg-app-tan.vercel.app/?slug=anna_ivanova_5442
  → клиент видит ТОЛЬКО этого мастера
```

### Ключевые решения

| Вопрос | Решение | Почему |
|--------|---------|--------|
| Один бот или много? | **Один бот** + deep links | Каждый мастер получает уникальную ссылку. Клиент видит только своего мастера |
| Оплата подписки | **ЮKassa** | Рубли, без комиссии Telegram Stars (~30%). Привычные карты для РФ |
| Админка мастера | **Mini App** (не бот-диалог) | Формы удобнее чем 25 сообщений для 5 услуг |
| Хранение фото | **Supabase Storage** | Бесплатно 1 GB, CDN, WebP |
| Хостинг | **Vercel** (Mini App + API + webhook) | Всё в одном месте, бесплатный HTTPS |
| Бот как консультант | **Автоответы + пересылка** (PRO) | Клиент спрашивает → бот отвечает (цены, адрес, слоты). Не распознал → пересылает мастеру |

## 2. Модель данных (Supabase / PostgreSQL)

> Схема уже создана в `supabase/001_schema.sql`. Ниже — описание с учётом доработок.

### masters — профиль мастера

```sql
CREATE TABLE masters (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  telegram_id   BIGINT UNIQUE NOT NULL,
  username      TEXT,
  slug          TEXT UNIQUE NOT NULL,       -- "anna_ivanova_5442"
  name          TEXT NOT NULL,
  avatar_url    TEXT,
  speciality    TEXT,
  experience    TEXT,
  address       TEXT,
  phone         TEXT,

  -- Настройки темы (PRO)
  theme         JSONB DEFAULT '{}',         -- {"accent":"#ff6b9d","style":"rounded"}

  -- Подписка
  plan          TEXT DEFAULT 'free' CHECK (plan IN ('free','pro')),
  plan_expires  TIMESTAMPTZ,

  total_bookings INT DEFAULT 0,
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now()
);
```

### services — услуги мастера

```sql
CREATE TABLE services (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id   UUID NOT NULL REFERENCES masters(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,
  description TEXT,
  price       INT NOT NULL CHECK (price >= 0),
  duration    INT NOT NULL CHECK (duration > 0),
  emoji       TEXT DEFAULT '💅',
  sort_order  INT DEFAULT 0,
  is_active   BOOLEAN DEFAULT true,
  created_at  TIMESTAMPTZ DEFAULT now()
);
-- Лимит FREE: max 5 активных — проверяется на уровне API
```

### portfolio — фото работ

```sql
CREATE TABLE portfolio (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id   UUID NOT NULL REFERENCES masters(id) ON DELETE CASCADE,
  image_url   TEXT NOT NULL,          -- URL в Supabase Storage
  caption     TEXT,
  sort_order  INT DEFAULT 0,
  created_at  TIMESTAMPTZ DEFAULT now()
);
-- Лимит FREE: max 5 фото — проверяется на уровне API
```

### schedule — расписание

```sql
CREATE TABLE schedule (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id     UUID NOT NULL REFERENCES masters(id) ON DELETE CASCADE,
  day_of_week   INT NOT NULL CHECK (day_of_week BETWEEN 0 AND 6),
  start_time    TIME NOT NULL,
  end_time      TIME NOT NULL,
  slot_duration INT DEFAULT 30,
  is_working    BOOLEAN DEFAULT true,
  UNIQUE(master_id, day_of_week)
);
-- Автоматически создаётся Пн-Сб 10:00-18:00 при регистрации (триггер)
```

### bookings — записи клиентов

```sql
CREATE TABLE bookings (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id       UUID NOT NULL REFERENCES masters(id),
  service_id      UUID NOT NULL REFERENCES services(id),
  client_tg_id    BIGINT NOT NULL,
  client_name     TEXT NOT NULL,
  client_username TEXT,
  date            DATE NOT NULL,
  time            TIME NOT NULL,
  end_time        TIME NOT NULL,
  status          TEXT DEFAULT 'pending'
                  CHECK (status IN ('pending','confirmed','cancelled','completed')),
  reminder_24h    BOOLEAN DEFAULT false,  -- PRO
  reminder_2h     BOOLEAN DEFAULT false,  -- PRO
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

### payments — оплаты подписок

```sql
CREATE TABLE payments (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id       UUID NOT NULL REFERENCES masters(id),
  amount          INT NOT NULL,             -- Сумма в рублях
  period          TEXT NOT NULL CHECK (period IN ('month','year')),
  status          TEXT DEFAULT 'pending'
                  CHECK (status IN ('pending','completed','refunded')),
  yukassa_id      TEXT,                     -- ID платежа ЮKassa
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

### messages — пересылка сообщений клиент↔мастер (PRO)

```sql
CREATE TABLE messages (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  master_id       UUID NOT NULL REFERENCES masters(id),
  client_tg_id    BIGINT NOT NULL,
  direction       TEXT NOT NULL CHECK (direction IN ('client_to_master','master_to_client')),
  text            TEXT NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

## 4. Роли и доступы

### Клиент
```
Видит:  профиль мастера, услуги, портфолио, свободные слоты
Может:  записаться, посмотреть свои записи, отменить запись (за 24ч)
Не может: менять данные мастера, видеть чужие записи
Идентификация: initData.user.id (из Telegram)
```

### Мастер (FREE)
```
Видит:  свои записи, свои услуги, свою статистику
Может:  CRUD до 5 услуг, загрузить до 5 фото, управлять расписанием,
        подтвердить/отменить записи, редактировать профиль
Не может: видеть данные других мастеров, менять тему, получать пересылку
Идентификация: initData.user.id === masters.telegram_id
```

### Мастер (PRO)
```
Всё что FREE, плюс:
  - Безлимит услуг и фото
  - Кастомная тема (цвета, стиль)
  - Напоминания клиентам (24ч, 2ч)
  - Пересылка сообщений от клиентов
  - Расширенная статистика (доход, популярные услуги, конверсия)
```

### Проверка initData (обязательно для защищённых API)

```javascript
import { createHmac } from 'crypto';

function validateInitData(initData, botToken) {
  const params = new URLSearchParams(initData);
  const hash = params.get('hash');
  params.delete('hash');

  const dataCheckString = [...params.entries()]
    .sort(([a], [b]) => a.localeCompare(b))
    .map(([k, v]) => `${k}=${v}`)
    .join('\n');

  const secretKey = createHmac('sha256', 'WebAppData')
    .update(botToken).digest();

  const checkHash = createHmac('sha256', secretKey)
    .update(dataCheckString).digest('hex');

  return checkHash === hash;
}
```

## 5. Бот: логика

### Команды

| Команда | Для кого | Действие |
|---------|----------|----------|
| `/start` | Новый пользователь | Выбор роли: «Я мастер» / «Я клиент» |
| `/start m_<slug>` | Клиент по deep link | Показывает профиль мастера + кнопка «Посмотреть услуги» (Mini App) |
| `/master` | Мастер | Дашборд: записи, статистика, кнопка «Админка» (Mini App) |
| `/help` | Все | Справка по командам |

### Регистрация мастера (в боте)

```
1. /start → «Я мастер»
2. Бот спрашивает имя → «Анна Иванова»
3. Бот спрашивает специальность → «Маникюр, Педикюр»
4. Бот создаёт запись в masters + расписание по умолчанию
5. Бот отправляет:
   - Ссылку для клиентов: t.me/tg_beautybot?start=m_anna_ivanova_5442
   - «Следующий шаг: добавьте услуги через /master»
```

### Бот как консультант

```
Клиент пишет боту текст (не команда):

1. Бот проверяет — привязан ли клиент к мастеру (по последней записи или deep link)
2. Если да:
   a. Ищет ключевые слова: "цена", "адрес", "время", "запись"
      → отвечает автоматически (данные мастера из БД)
   b. Не распознал → пересылает мастеру (если PRO)
      → мастер отвечает в боте → бот доставляет клиенту
3. Если нет привязки:
   → «Перейдите по ссылке от мастера, чтобы записаться»
```

### Уведомления

| Событие | Кому | Сообщение |
|---------|------|-----------|
| Новая запись | Мастеру | «Новая запись! Елена — Маникюр, 18 мар 14:00 [Подтвердить] [Отменить]» |
| Подтверждение | Клиенту | «Ваша запись подтверждена! Маникюр, 18 мар 14:00» |
| Отмена мастером | Клиенту | «Мастер отменил запись. Выберите другое время» |
| Отмена клиентом | Мастеру | «Клиент Елена отменил запись на 18 мар 14:00» |
| Напоминание 24ч (PRO) | Клиенту | «Завтра в 14:00 — Маникюр у Анны. Адрес: ...» |
| Напоминание 2ч (PRO) | Клиенту | «Через 2 часа — Маникюр у Анны» |
| Оплата успешна | Мастеру | «PRO активирован до 13 апр 2026!» |
| Подписка истекает | Мастеру | «PRO истекает через 3 дня. Продлить?» |

## 7. Оплата через ЮKassa

### Флоу

```
1. Мастер нажимает «Оформить PRO» в админке
2. Mini App → POST /api/payment/create { master_id, period: "month" }
3. API создаёт платёж в ЮKassa:
   - amount: 299.00 (месяц) или 2490.00 (год)
   - confirmation: { type: "redirect", return_url: "https://t.me/tg_beautybot" }
4. API возвращает redirect URL → Mini App открывает ссылку
5. Клиент оплачивает на странице ЮKassa
6. ЮKassa отправляет webhook → POST /api/payment/webhook
7. API проверяет подпись, активирует подписку:
   - masters.plan = 'pro'
   - masters.plan_expires = now() + 30 дней (или 365)
   - Запись в payments
8. Бот уведомляет мастера: «PRO активирован!»
```

### Конфигурация ЮKassa

```
Переменные окружения:
  YUKASSA_SHOP_ID=...
  YUKASSA_SECRET_KEY=...
  YUKASSA_WEBHOOK_SECRET=...  (для проверки подписи)
```
